Capabilities

让 agent 拥有真正的"长期记忆"

不是 cookie 里的临时状态，不是会被截断的 context window。 是一份属于 agent 的、跨设备持续累积的、可被搜索和共享的记忆。

云端自动同步

本地写、云端存。后台 daemon 实时把记忆推上云， 同时拉取远端的更新——双向无缝。WPS 云备份的 agent 心智。

语义搜索

服务端 sentence-transformers 自动 embed，sqlite-vec / pgvector 索引。 "用户偏好什么风格" 直接命中相关记忆，不用想关键词。

一个 Key = 一个身份

没有邮箱、没有密码、没有手机号、没有验证码。 32 字节 base58 key = 完整身份。Recovery code 让你丢了也能恢复。

分享给特定人

可撤销、可设过期、可设权限（读 / 读+搜 / 全部）的 share token。 别人凭 token 消费你的记忆 timeline + 语义搜索，无需注册。

Event Sourcing 协议

所有变更写 append-only log。天然支持回放、时间旅行、审计。 Memory / asset 都是事件的查询视图——开放、简单、可重建。

开源 + 开放数据格式

Apache 2.0，自部署免费。Event 走 JSON 自描述， 不绑死 LangChain / AutoGen / Cursor / 任何 agent runtime。

3 steps

3 分钟接入

从零到 agent 拥有云端记忆，只需要 3 行命令。

1

安装

pip install agentcloud 或一行安装脚本：
curl -fsSL install.sh | bash

2

注册

agentcloud register
拿到一个 master key（保存好）+ recovery code。

3

开始

agentcloud sync daemon --start
你的 agent 写记忆，自动同步到云端。

代码示例

从你的 agent 代码里直接调用——任何 Python agent 都能用。

# register (once)
from agentcloud import AgentCloud
ac = AgentCloud.register("http://your-server:8000", label="my-agent")
ac.save()  # credentials → ~/.agentcloud/credentials.json

# start background sync (daemon thread)
ac.sync.daemon_start(push_interval=1.0, pull_interval=5.0)

# write memory (auto-pushed by daemon)
ac.memory.add("用户喜欢简洁回答", type="preference", tags=["user:zhang"])

# semantic search
hits = ac.memory.search("用户偏好什么", top_k=5)
for h in hits:
    print(h.score, h.content)

# share with a friend (read-only token, expires in 1 day)
token, info = ac.share.create(permissions="read_memory", expires_in=86400)
# 朋友: 凭 token 读你的记忆，无需注册
shared = AgentCloud.connect_share(token, server_url="http://...")
their_items = shared.timeline(limit=20)
